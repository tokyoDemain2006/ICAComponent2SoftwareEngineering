Adaptor Pattern

Actuators

  • Imagine, You have an control system managing three different types of actuators:
    • Hydraulic Actuator: Uses Method ActivatePressure(int level).
    • Pneumatic Actuator: Uses Method StartAirFlow(float psi).
    • Electric Actuator: Uses Method EnableVoltage(double volts, bool safetyCheck).
  • Each as a different signature, but what is wanted is a system with a common interface for actuators so the control software can interact with them uniformly, without worrying about different signatures.

Step 1

  • Define a Common Interface
  • define a standard interface for actuators so the system can use them consistently.

public interface IActuator

{

void Activate(int intensity);

}

Step 2

  • Create Adapters for Each Actuator Type

// Existing class (incompatible)

public class HydraulicActuator

{

public void ActivatePressure(int level)

{

Console.WriteLine($"Hydraulic Actuator: Applying {level} pressure.");

}

}

// Adapter

public class HydraulicAdapter : IActuator

{

private HydraulicActuator _hydraulicActuator = new HydraulicActuator();

public void Activate(int intensity)

{

_hydraulicActuator.ActivatePressure(intensity);

}

}

Step 3: The Controller

class ActuatorController

{

private IActuator _actuator;

public ActuatorController(IActuator actuator)

{

_actuator = actuator;

}

public void Trigger(int intensity)

{

_actuator.Activate(intensity);

}

}

Principles to Patterns

SOLID Principle

How the Adapter Pattern Implements It

Single Responsibility (SRP)

The adapter only converts one form into another.

Open/Closed (OCP)

You can add new adapters without modifying existing code.

Liskov Substitution (LSP)

Any adapter implementing IActuator can replace another.

Dependency Inversion (DIP)

ActuatorController depends on IActuator, not concrete actuator classes.

Facade Pattern

  • a structural design pattern that provides a simplified, high-level interface to a complex system of classes, libraries, or APIs.
  • Instead of exposing multiple components with intricate dependencies, the Facade acts as a single entry point that orchestrates interactions between subsystems.

Typical Applications

    • Complex APIs & Libraries
      • Facade provides a cleaner interface over third-party SDKs or lower-level system calls.
      • Example: A PaymentFacade simplifies multiple payment gateways (PayPal, Stripe, Bank API).
    • Subsystem Management
      • When a system has multiple interdependent modules, a Facade can manage their interactions.
      • Example: A MediaPlayerFacade controlling audio, video, and subtitles in a media player.
    • Legacy System Integration
      • Facade bridges old, complicated code with a modern, user-friendly API.
      • Example: Wrapping a C-based legacy API in a clean C# or Java API.
    • Microservices & Distributed Systems
      • Facade can act as an API Gateway, handling requests to multiple microservices.
      • Example: OrderFacade in an e-commerce system handling inventory, payments, and shipping via separate microservices.

public class MachineFacade

{

private List<ISensor> _sensors;

private List<IActuator> _actuators;

public MachineFacade()

{

// Initialise sensors

_sensors = new List<ISensor>

{

new TemperatureAdapter(),

new PressureAdapter(),

new HumidityAdapter()

};

// Initialise actuators

_actuators = new List<IActuator>

{

new HydraulicAdapter(),

new PneumaticAdapter(),

new ElectricAdapter()

};

}

// Read all sensors and return their values

public void ReadAllSensors()

{

Console.WriteLine("Reading sensor data...");

foreach (var sensor in _sensors)

{

Console.WriteLine($"Sensor Reading: {sensor.GetReading()}");

}

}

// Activate all actuators at a given intensity

public void ActivateAllActuators(int intensity)

{

Console.WriteLine($"Activating all actuators with intensity: {intensity}");

foreach (var actuator in _actuators)

{

actuator.Activate(intensity);

}

}

// Start the machine by initializing all components

public void StartMachine()

{

Console.WriteLine("Starting the machine...");

ReadAllSensors();

ActivateAllActuators(50);

Console.WriteLine("Machine started successfully!");

}

}

Model

Adaptor & Facade

  • The Adapter pattern allows incompatible interfaces to work together by acting as a bridge between two components.
  • converts the interface of a class into one that a client expects.
  • The Facade pattern provides a simplified interface to a complex system, hiding the details of subsystems.
  • does not change functionality but organises and reduces complexity.

Pattern

Purpose

How It’s Used Here
